Major overhaul of visionctl agent system with improved LLM interaction, better error handling, and comprehensive logging

## Summary of Changes

### Core Improvements
1. **Comprehensive Logging System**
   - Added `tracing` crate to both `inputctl` and `visionctl`
   - Replaced all `eprintln!` calls with structured logging (info, debug, warn, error, trace)
   - Added verbose/quiet flags to CLI for log level control
   - Log levels: error (quiet), warn (default), info (verbose), debug (very verbose), trace (most verbose)

2. **Agent System Redesign**
   - Complete rewrite of agent planning and execution logic
   - Better error handling with automatic replanning when tools fail
   - Added scene notes for tracking UI state across iterations
   - Improved cursor position tracking with grid metadata
   - Better tool call management with proper ID tracking

3. **Tool System Improvements**
   - Replaced template matching tools with icon-based tools
   - `move_to_cell` now supports optional pixel offsets for precision
   - Added `scene_note` tool for documenting UI state
   - Added `stuck` tool for indicating when agent needs help
   - Improved `move_to_icon` to automatically move cursor to found icons
   - Better tool response handling with cursor status feedback

4. **Code Quality**
   - Removed debug print statements throughout codebase
   - Added proper error handling for all tool calls
   - Better state management in agent execution loop
   - Improved message history tracking
